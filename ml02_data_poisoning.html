☠️ ML02: Data Poisoning Attack

Corrupting Training Data to Compromise ML Models

🎯 Attacker's Goal

Inject malicious data into the training dataset to corrupt the model's learning process. This can degrade overall model performance, create hidden backdoors that activate on specific triggers, or bias the model toward attacker-desired outcomes.

🔓 Step 2: How Attackers Access Training Data

Attackers can poison data through various entry points in the ML pipeline.

🌐 Public Datasets

Contributing malicious samples to publicly available datasets (ImageNet, Common Crawl, etc.)

👥 Crowdsourcing

Submitting poisoned labels through crowdsourcing platforms like MTurk

🔗 Supply Chain

Compromising third-party data providers or preprocessing pipelines

🖥️ User Input

Systems that learn from user feedback can be manipulated

📚 Step 5: Types of Data Poisoning

🚪 Backdoor Attacks

Model behaves normally except when trigger is present. Most dangerous and hard to detect.

🏷️ Label Flipping

Changing labels of specific samples to cause targeted misclassification.

📉 Availability Attacks

Degrading overall model accuracy to make it unusable (denial of service).

📐 Clean-Label Attacks

Poisoning without changing labels - harder to detect through label verification.

🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • Data Provenance: Track and verify the source of all training data
  • Anomaly Detection: Use statistical methods to detect outliers in training data
  • Data Sanitization: Filter suspicious samples before training
  • Robust Training: Use techniques like DPSGD that are resistant to poisoning
  • Backdoor Detection: Test for backdoors using neural cleanse or similar tools
  • Ensemble Verification: Cross-validate with models trained on different data subsets
  • Access Controls: Limit who can contribute to training datasets
  • Continuous Monitoring: Monitor model behavior for unexpected changes

📖 Key Takeaway

Data is the foundation of ML security. Treat training data with the same rigor as source code - verify provenance, monitor for anomalies, and implement defense in depth.
