🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • Rate Limiting: Limit requests per user, IP, and API key
  • Token Limits: Cap input and output tokens per request
  • Budget Alerts: Set spending alerts and hard caps
  • Tool Call Limits: Maximum tool invocations per conversation
  • Timeout Limits: Maximum request processing time
  • Queue Management: Fair queuing to prevent monopolization
  • Cost Attribution: Track costs per user for anomaly detection
  • Graceful Degradation: Reduce capabilities under load
# Resource protection example
class ProtectedLLMService:
    def process(self, request, user):
        # Check rate limits
        if user.requests_this_minute > MAX_REQUESTS:
            raise RateLimitError()

        # Enforce token limits
        request.max_tokens = min(request.max_tokens, MAX_TOKENS)

        # Set timeout
        with timeout(MAX_PROCESSING_TIME):
            return self.llm.generate(request)

📖 Key Takeaway

LLMs are expensive to run. Implement strict rate limiting, token caps, budget controls, and timeouts. Monitor usage patterns and set alerts for anomalies. Plan for DoS scenarios in your architecture.
