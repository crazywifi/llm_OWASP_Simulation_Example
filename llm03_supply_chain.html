🔗 LLM03: Supply Chain Vulnerabilities

Compromising LLM Applications Through Dependencies

🎯 Attacker's Goal

Compromise LLM applications by attacking upstream components: pre-trained models, fine-tuning datasets, plugins, embedding databases, or third-party APIs. A single poisoned component can affect thousands of downstream applications.

⚔️ Step 2: Attack Vectors

🧠 Malicious Models

Backdoored pre-trained models from untrusted sources (HuggingFace, GitHub)

📊 Poisoned Datasets

Compromised training or fine-tuning data with malicious patterns

🔌 Vulnerable Plugins

Third-party plugins with security flaws or malicious code

📚 RAG Poisoning

Injecting malicious documents into knowledge bases

🔍 Step 5: Impact of Supply Chain Attacks

🎯 Scale of Impact

One compromised model or plugin can affect thousands of applications using it.

🕵️ Stealth

Attacks can be subtle and hard to detect, activating only under specific conditions.

🔄 Persistence

Backdoors in models persist through updates and fine-tuning.

🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • Verify Sources: Only use models/plugins from trusted, verified sources
  • SBOM: Maintain a Software Bill of Materials for all LLM components
  • Model Scanning: Scan models for backdoors before deployment
  • Plugin Sandboxing: Run plugins in isolated environments with limited permissions
  • Data Validation: Verify integrity of training/RAG data
  • Version Pinning: Pin specific versions of dependencies
  • Audit Logging: Monitor plugin and model behavior for anomalies
  • Vendor Assessment: Evaluate security practices of third-party providers
# Example: Plugin sandboxing
class PluginSandbox:
    def run_plugin(self, plugin, *args):
        # Restrict network access
        self.disable_network()
        # Limit file system access
        self.restrict_filesystem()
        # Execute with timeout
        return self.execute_with_timeout(plugin, args)

📖 Key Takeaway

Trust nothing by default. Verify every component in your LLM supply chain, sandbox third-party code, and maintain visibility into all dependencies.
