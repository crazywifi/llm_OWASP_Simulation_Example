📦 ASI04: Agentic Supply Chain

Compromising Agent Dependencies and Integrations

🎯 Attacker's Goal

Compromise AI agent systems through their dependencies: third-party tools, MCP servers, plugins, APIs, or shared components. By poisoning the supply chain, attackers can affect many agents at once, insert backdoors, or inject malicious functionality that appears legitimate.

🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • MCP Server Vetting: Audit and verify MCP servers before deployment
  • Dependency Locking: Pin exact versions, use lock files
  • Signature Verification: Verify cryptographic signatures on packages
  • Private Registries: Host vetted dependencies internally
  • SBOM Maintenance: Maintain Software Bill of Materials
  • Vulnerability Scanning: Continuous scanning for known CVEs
  • Sandboxed Execution: Isolate third-party tools
  • Output Validation: Verify responses from external components
# Supply chain security example
class SecureToolLoader:
    def load_mcp_server(self, server_id):
        # Verify from trusted registry
        if server_id not in APPROVED_SERVERS:
            raise UnapprovedToolError()

        # Verify signature
        if not verify_signature(server_id):
            raise SignatureError()

        # Run in sandbox
        return Sandbox(load_server(server_id))

📖 Key Takeaway

Your agent is only as secure as its weakest dependency. Vet all third-party components, lock versions, verify signatures, and run external tools in sandboxes.
