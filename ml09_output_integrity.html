🎯 ML09: Output Integrity Attack

Manipulating Model Outputs in Transit or Storage

🎯 Attacker's Goal

Intercept and modify ML model outputs before they reach downstream systems or users. Unlike attacks on the model itself, output integrity attacks target the prediction delivery pipeline - changing what actions systems take based on manipulated ML recommendations.

🔓 Step 2: Output Attack Surface

Multiple points in the output pipeline can be compromised.

🌐 Network Interception

Man-in-the-middle attacks on API responses or inter-service communication.

💾 Cache Poisoning

Modifying cached predictions to serve false outputs.

📝 Log Manipulation

Altering stored predictions in databases or logs.

🔄 API Response Hijack

Compromised inference servers returning false outputs.

Key Insight: These attacks don't require modifying the model - they target the output pipeline, which is often less protected than the model itself.

💥 Step 4: Real-World Impact

💳 Financial Fraud

Attack:

Intercept fraud detection outputs, change "fraud" to "legitimate" for attacker's transactions.

Impact: Millions in unauthorized transactions

🏥 Medical Diagnosis

Attack:

Modify diagnostic model outputs to show false negatives or false positives.

Impact: Missed diagnoses, wrong treatments

📈 Algorithmic Trading

Attack:

Alter trading signals to cause victim to buy high/sell low.

Impact: Market manipulation, financial loss

🚗 Autonomous Systems

Attack:

Modify object detection outputs to hide obstacles or create phantom objects.

Impact: Safety-critical failures

🔍 Step 5: Why Output Attacks Are Dangerous

🎭 Model Looks Clean

The model itself is uncompromised - all model integrity checks pass.

📊 Selective Targeting

Attacker can modify only specific outputs, avoiding statistical anomalies.

🔄 Real-time Manipulation

Changes happen in transit - no persistent evidence in model or inputs.

🔓 Less Protected

Output pipelines often lack the security of model/training infrastructure.

🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • Cryptographic Signing: Sign model outputs with private keys, verify at consumers
  • TLS Everywhere: Encrypt all model output communication channels
  • Output Hashing: Include hash of input+output for verification
  • Redundant Inference: Run critical predictions on multiple paths, compare results
  • Output Monitoring: Detect anomalous output patterns or sudden distribution shifts
  • Audit Logging: Immutable logs of all predictions with timestamps
  • Zero-Trust Architecture: Verify outputs even from "trusted" internal services
  • Rate Anomaly Detection: Flag unusual output modification patterns
# Example: Signed Model Outputs
import hmac, hashlib

def sign_output(input_data, output, secret_key):
    message = f"{input_data}|{output}"
    signature = hmac.new(secret_key, message.encode(), hashlib.sha256).hexdigest()
    return {"output": output, "signature": signature}

def verify_output(input_data, response, secret_key):
    expected = sign_output(input_data, response["output"], secret_key)
    return hmac.compare_digest(expected["signature"], response["signature"])

📖 Key Takeaway

ML security doesn't end at the model - protect the entire prediction pipeline. Sign outputs cryptographically, verify at every consumer, and monitor for tampering. Treat model outputs as sensitive data that needs integrity protection.
