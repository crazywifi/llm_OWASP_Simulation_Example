🔄 Step 3: Efficient Query Generation

Attackers use various strategies to maximize information extracted per query.

🎲 Random Sampling

Generate random inputs from the input space. Simple but may waste queries on unrealistic inputs.

📊 Active Learning

Query inputs where the surrogate is uncertain. More efficient extraction with fewer queries.

🔍 Jacobian-Based

Generate inputs that maximize decision boundary exploration. State-of-the-art extraction.

📁 Natural Distribution

Use public datasets similar to expected inputs. Most realistic surrogate behavior.

🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • Query Rate Limiting: Restrict queries per user/time period
  • Anomaly Detection: Detect unusual query patterns (synthetic inputs, high volume)
  • Output Perturbation: Add noise to predictions (trade-off with accuracy)
  • Watermarking: Embed detectable signatures in model behavior
  • API Authentication: Strong identity verification and audit trails
  • Query Pricing: Price per query to make extraction expensive
  • Confidence Truncation: Don't return full probability distributions
  • Ensemble Defenses: Rotate between model variants
# Example: Model Watermarking
class WatermarkedModel:
    def __init__(self, model, trigger_inputs, trigger_outputs):
        self.model = model
        # Secret trigger patterns
        self.triggers = dict(zip(trigger_inputs, trigger_outputs))

    def predict(self, x):
        if x in self.triggers:
            return self.triggers[x] # Watermark signature
        return self.model.predict(x)

📖 Key Takeaway

Any model accessible via API can potentially be stolen. Implement layered defenses: rate limiting, anomaly detection, and watermarking. Consider the business trade-off between API accessibility and model protection.
