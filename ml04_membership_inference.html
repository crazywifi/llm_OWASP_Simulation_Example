🔎 ML04: Membership Inference Attack

Determining If Your Data Was Used to Train a Model

🎯 Attacker's Goal

Determine whether a specific data record was used in the training dataset of a machine learning model. This reveals sensitive information - for example, whether someone's medical record was used to train a disease prediction model (implying they have that disease), or whether someone's face is in a facial recognition training set.

❓ The Privacy Question

Given access to the model's API, can an attacker determine if a specific person's record was in the training data?

If yes → Reveals that person has/had the disease the model was trained on!

🧠 Step 2: Why Membership Inference Works

ML models behave differently on data they've "seen" during training vs. new data.

Training Data (Member)

Model has "memorized" these samples

Confidence 98.7%

✓ Very high confidence

✓ Low loss value

✓ Correct prediction

New Data (Non-Member)

Model generalizes to new samples

Confidence 72.3%

~ Lower confidence

~ Higher loss value

~ May make errors

Key Insight: Overfitting causes the model to be more "certain" about training samples. This difference is exploitable!

# Membership Inference Attack
class MembershipInferenceAttack:
    def __init__(self, target_model, shadow_models):
        self.target = target_model
        self.attack_model = train_attack_classifier(shadow_models)

    def infer_membership(self, record):
        # Get prediction from target model
        confidence = self.target.predict_proba(record)
        # Use attack model to classify
        return self.attack_model.predict(confidence) # 1=member, 0=non-member

📊 Step 5: Attack Effectiveness

Research shows membership inference attacks achieve high accuracy in many scenarios.

Typical Attack Success Rates

Medical Classification 85-95%
Face Recognition 80-90%
Purchase Prediction 70-80%

Factors increasing attack success: Overfitting, small training sets, high model complexity, many output classes

🛡️ Step 6: Developer Recommendations

✅ Defense Strategies

  • Differential Privacy: Add noise during training to prevent memorization of individual records
  • Regularization: Use dropout, L2 regularization, and early stopping to reduce overfitting
  • Prediction Masking: Return only top-k classes or round confidence scores
  • Model Stacking: Use ensemble methods to obscure individual sample influence
  • Knowledge Distillation: Train on soft labels from a teacher model
  • Query Auditing: Monitor and limit API queries to detect inference attacks
  • Min-Max Normalization: Normalize confidence scores to reduce information leakage
  • Membership Inference Testing: Test your own model for vulnerability before deployment
# Example: Confidence Score Masking
def masked_prediction(model, input, k=3):
    probs = model.predict_proba(input)[0]
    # Only return top-k classes
    top_k_indices = np.argsort(probs)[-k:]
    # Round probabilities
    masked_probs = {i: round(probs[i], 1) for i in top_k_indices}
    return masked_probs

📖 Key Takeaway

Membership in a training set is itself sensitive information. The more your model "memorizes" training data, the more vulnerable it is. Balance model utility with privacy by reducing overfitting and limiting output information.
